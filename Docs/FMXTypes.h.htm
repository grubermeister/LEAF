FMXTypes.h

The FMXStyles.h header contains two main classes. One is QuadChar, the other is Locale. Both have their autopointer classes: QuadCharAutoPtr and LocalePtr.

Class QuadCharAutoPtr is used to store a QuadChar, which consists of four char (byte) values. It is used to store and set the ID of your plug-in, but is also used as Binary Data stream type. We will get into this during our discussion of the FMXBinaryData.h header, which also mentions several examples. For now, we can just work with the definition that says it contains 4 bytes, or chars. The autopointer has four constructors that create an autopointer and a QuadChar of which the address is saved. Three of those are defined in this header. The fourth is derived from class auto_ptr and sets the new pointer to the value of an existing one that you pass to it. The other three constructors are pretty straightforward:

class QuadCharAutoPtr : public std::auto_ptr<QuadChar>
{
    typedef QuadCharAutoPtr UpCaster;//not discussed here; see FMXExtern.h
public:
    inline QuadCharAutoPtr ();
    inline QuadCharAutoPtr ( char c0, char c1, char c2, char c3 );
    inline QuadCharAutoPtr ( const QuadChar &value );
};

The first constructor does not set any values. It creates an autopointer that points to a QuadChar containing four chars set to NULL. To the second QuadCharAutoPtr constructor you have to pass four chars. The last constructor takes a QuadCharand sets the autopointer to a copy of this object. Now that we have an autopointer pointing to a QuadChar object, what can we do with it? Well, the class QuadChar has the following methods:

  1. Ten operator functions, of which
    1. One assignment operator = function
    2. Two array operator [] functions (subscript operator functions)
    3. Six comparison operator functions
    4. The delete operator which you don't need to use because the autopointer takes care of that
  2. Two methods to set and retrieve a MacType

The assignment operator function makes sure you can assign one QuadChar to another.

?
1
2
3
fmx::QuadCharAutoPtr QAP;
fmx::QuadCharAutoPtr QAP2;
*QAP = *QAP2;

Now QAP's object is removed and it points to the same object as QAP2. Note that if you now create a third autopointer and assign its object to QAP2, the old object QAP2 pointed to is removed (and replaced), and the original QAP does not point to anything anymore.

The two subscript operator functions enable you to read one char at a time from the QuadChar. To read from a QuadChar you can do this:

?
1
2
3
4
fmx::QuadCharAutoPtr QAP('a','b','c','d');
fmx::uchar CH = QAP->operator[](2); // CH has value 'c'
//or an easier notation:
fmx::uchar CH2 = (*QAP)[2]; //first dereference the autopointer

The six comparison operators are used to compare two QuadChar objects. The operator== and operator!= return true or false when all their characters match or not. The larger than and smaller than operators start by checking the first character. Only when they are the same, it will look at the second character, etc. If used as byte values, an object with values 123-7-88-2 is larger than an object with values 123-6-135-99. An object with values 255-44-254-255 is smaller than an object with 255-44-255-1.

The GetMacType and SetMacType methods are not documented anywhere and their use is a mystery. The MacType is set to 0 when you create a new QuadChar. Set it tio something else if you like, but don't expect any result.

The next class in the FMXTypes.h header is Locale. It is created by using a construcor from the class LocaleAutoPtr. The latter has the following constructors.

class LocaleAutoPtr : public std::auto_ptr<Locale>
{
        typedef LocaleAutoPtr   UpCaster;
public:
    inline LocaleAutoPtr ( Locale::Type inputType = Locale::kType_System );
    inline LocaleAutoPtr ( const Locale &copyConstruct );
};

The first constructor takes a Type as argument. The second simply takes another Locale object and copies that. The Type passed to the first constructor is a value from an enumeration with the identifier Type. When no argument is passed the default is used, which is kType_System, the default to which the OS is set. A user can change the settings of the OS, which will also change the locale that is set with kType_System. There are quite a few  Types to which you can set a Locale.
